    • Centos7下安装postgres

    • 验证postgres是否安装成功或者查看系统是否有postgres：
    • #sudo rpm -aq| grep postgres
    • 卸载已有的postgres:
    • #yum remove postgresql*

    • 安装postgres [postgres下载地址: https://yum.postgresql.org/repopackages.php]
      以postgres6为例:

    • Step1,安装PostgreSQL源:

    • #rpm -Uvh https://download.postgresql.org/pub/repos/yum/9.6/redhat/rhel-7-x86_64/pgdg-centos96-9> * .6-3.noarch.rpm
    • Step2,执行安装命令:
    • sudo yum update

    • sudo yum install postgresql96-server postgresql96-contrib

    • Step3, 验证是否安装成功
      sudo rpm -aq| grep postgres
      执行结果如下:
      postgresql96-9.6.2-2PGDG.rhel7.x86_64
      postgresql96-server-9.6.2-2PGDG.rhel7.x86_64
      postgresql96-libs-9.6.2-2PGDG.rhel7.x86_64
      postgresql96-contrib-9.6.2-2PGDG.rhel7.x86_64

    • Step4, 初始化数据库

    • sudo /usr/pgsql-9.4/bin/postgresql94-setup initdb

    • 在初始化数据库时可以指定参数 –PGDATA=“/data”，该参数是用于指明数据库的数据文件的存放路径，默认是在/var/lib/pgsql/9.4/data路径下

    • Step5, 启动服务并设置为开机启动
      [root@wenddy ~]# sudo systemctl enable postgresql-9.6
      Created symlink from /etc/systemd/system/multi-user.target.wants/postgresql-9.6.service to /usr/lib/systemd/system/postgresql-9.6.service.

    [root@wenddy ~]# sudo systemctl start postgresql-9.6

    • Step6, 开放防火墙端口

    • sudo firewall-cmd –permanent –add-port=5432/tcp
      success

    • sudo firewall-cmd –permanent –add-port=80/tcp
      success
    • sudo firewall-cmd –reload
      Success
    • Step7, 访问PostgreSQL
      [root@wenddy ~]# su - postgres
      Last login: Tue Mar 14 13:08:14 CST 2017 on pts/0
      -bash-4.2$ psql
      输入命令psql将看到PostgrSQL的版本信息。
      psql (9.6.2)
      Type “help” for help.
    • Step8,设置postgres用户密码
      postgres=# \password postgres
      Enter new password:
      Enter it again:
      输入su – postgres
      输入 psql,进行pgsql操作:

    4.1 Postgresql控制台命令:

    • §\q：命令退出控制台
    • §\password：为XXX用户设置一个密码。
    • §\h：查看SQL命令的解释，比如\h select。
    • §\?：查看psql命令列表。
    • §\l：列出所有数据库。
    • §\c [database_name]：连接其他数据库。
    • §\d：列出当前数据库的所有表格。
    • §\d [table_name]：列出某一张表格的结构。
    • §\du：列出所有用户。
    • §\e：打开文本编辑器。
    • §\conninfo：列出当前数据库和连接的信息

    4.2 查看pgsql安装路径:

    ps -ef|grep postgres

    4.3 PostgresSQL 数据库配置实现远程访问

    • 安装PostgreSQL数据库之后，默认是只接受本地访问连接。如果想在其他主机上访问PostgreSQL数据库服务器 *，就需要进行相应的配置。只需要修改/var/lib/pgsql/9.6/data/目录下的pg_hba.conf和postgresql.conf。
    • pg_hba.conf：配置对数据库的访问权限，
    • postgresql.conf：配置PostgreSQL数据库服务器的相应的参数。
    • 修改pg_hba.conf
      这个文件最后有一个列表，它决定了分派了每一个用户的权限，以及认证方式。格式是“Type Database User Address Method”，要注意的是method最好写md5

    1. TYPE DATABASE USER ADDRESS METHOD

    2. “local” is for Unix domain socket connections only

    3. local all all peer

    4. IPv4 local connections:

    5. host all all 127.0.0.1/32 trust
    6. #host all all 0.0.0.0/0 md5
    7. host all all 0.0.0.0/0 trust

    8. IPv6 local connections:

    9. host all all ::1/128 ident

    10. Allow replication connections from localhost, by a user with the

    11. replication privilege.

    12. #local replication postgres peer
    13. #host replication postgres 127.0.0.1/32 ident
    14. #host replication postgres ::1/128 ident
    • 修改postgresql.conf文件
      将数据库服务器的监听模式修改为监听所有主机发出的连接请求。
      将listen_addresses前的#去掉，并将listen_addresses =’localhost’改成listen_addresses = ‘*’
      查看：[xxx@xxx ~]# ps -ef|grep postgres
    1. listen_addresses = ‘*’ # what IP address(es) to listen on;

    2. comma-separated list of addresses;

    3. defaults to ‘localhost’; use ‘*’ for all

    4. (change requires restart)

    5. #port = 5432

    • 防火墙
      进入/usr/lib/firewalld/services/,若有pgsql的xml文件，则
      firewall-cmd –permanent –add-service=xml文件名
      firewall-cmd –reload

    • 若没有则新建,xml文件内容:

    • 然后执行

    • firewall-cmd –permanent –add-service=xml文件名
    • firewall-cmd –reload
    • 重启服务以使设置生效

    • service postgresql-9.2 restart
