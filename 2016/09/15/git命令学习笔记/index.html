git 初始化

  • git config –global user.email userEmail
  • git config –global user.name userName

repositary 操作

1. 添加仓库

  • git remote add 别名 “url”
  • eg: git remote add origin originUrl

2. 移除仓库

  • git remote remove origin/upstream

3. 更新仓库地址

  • git remote set-url origin newOriginUrl

4. 查看他人仓库

  • git remote add <somebody’s git>
  • git remote rm <somebody’s git>

5. clone

  • git clone gitUrl

6. 查看仓库详细信息

  • git remote -v

7. view repositary history

  • gitk –all&

branch 操作

1. 创建分支

  • git branch feature1
  • check out feature1
  • commit feature1
  • push origin feature1

2. 删除本地分支

  • git branch -d feature1
  • git branch -D feature1

3. 删除远程分支

  • git push origin –delete feature1

4. 切换分支

  • git checkout master/develop （切换到master/develop分支）

5. 查看所有分支

  • git branch -v

tag操作

1. 本地新增tag

  • git tag -a tagName -m “comments”

2. 推送指定tag

  • git push origin tagName

3. 推送所有本地新增标签

  • git push origin –tags

4. 删除tag

  • git tag -d tagName

5. 同步删除远程分支上的tag

  • git push upstream develop :tag名称
    (注：tag名称前没有”:”是新增,有”:”是删除)

6. 查看tag

  • git tag

push代码

1. 提交代码

  • git status
  • git add fileName
  • git commit -m “comments”

2. 修改上一次提交信息/内容

  • git commit –amend -m “msg” //修改上一次的提交信息
  • git commit –amend //修改上一次提交的内容

保留本地修改，更新最新代码

  • git stash
  • pull upstream
  • git stash pop

删除stash

  • git stash drop

develop cherry-pick到master上冲突

  • git checkout master
  • git status，查看冲突文件
  • 手动解决冲突文件
  • git add 修改的文件
  • git status查看
  • git commit :在comment后添加 resolved,输入”:wq”保存并退出

git merge

origin 下的branchA merge 到develop

  • git checkout develop
  • git stash
  • git merge origin/branchA
  • git stash pop
  • git push origin develop -f
    （force push）

update repositary

git remote update
git pull upstream develop

git add 的撤销操作

git reset HEAD file
